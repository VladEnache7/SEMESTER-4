### üìö Key Topics Breakdown

#### Part I: Analysis & Modeling
*   **UML (Unified Modeling Language):** The standard language for visualizing system architecture.
*   **Requirements Elicitation:** Interviewing stakeholders, writing Use Cases, identifying functional vs. non-functional requirements.
*   **Diagrams:**
    *   **Structural:** Class Diagrams (Static view).
    *   **Behavioral:** Sequence, Activity, and State Machine Diagrams (Dynamic view).

#### Part II: Design & Architecture
*   **System Design:** High-level architecture, subsystem decomposition.
*   **Object Design:** Detailed class design, interface specification.
*   **Design Patterns:** Reusing standard solutions (e.g., Singleton, Factory, Observer).
*   **Design by Contract:** Using assertions, pre-conditions, and post-conditions to ensure reliability.

#### Part III: Management & Quality
*   **Agile/Scrum:** Iterative development, sprints, backlogs.
*   **Testing:** Black-box vs. White-box testing, Unit tests, Integration tests.
*   **Configuration Management:** Version control (Git), build systems, and release management.
